# Задача: Асинхронный агрегатор данных о пользователях

### Контекст
Вы разрабатываете сервис для HR‑отдела. Нужно собрать полную информацию о сотруднике из трёх разных микросервисов и вернуть единый отчёт.

### Условия

Есть 3 асинхронных источника данных (имитируются методами‑заглушками):
1. fetchBasicInfo(id) — базовая информация (имя, email);
2. fetchEmploymentData(id) — данные о трудоустройстве (должность, отдел);
3. fetchSalaryInfo(id) — зарплатные данные.

Если нет базовой информации о пользователе (fetchBasicInfo) - то запросы в оставшиеся два сервиса не посылаем.
При этом если какой-то из запросов к сервисам fetchEmploymentData или fetchSalaryInfo завершился с ошибкой, то формирование отчета не должно прерываться. 

Необходимо заполнить соответсвующие поля в итоговом отчете пропусками.
Итоговый отчёт должен содержать:

```java
class EmployeeReport {
    String name;
    String email;
    String position;
    String department;
    BigDecimal salary;
    LocalDateTime fetchedAt; // текущее время`
}
```
